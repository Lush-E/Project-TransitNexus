# Handover Memo (2026-03-27)

This memo summarizes rendering fixes, selection regressions, viaduct area erase compatibility, and the new line-color preset window work completed today.

## Completed Today

### 1) Crossing / Connection Circle Visibility and Masking

- Unified visibility behavior between:
  - intersection correction circles
  - connection node circles
- Reworked overlap handling with higher viaduct areas to avoid black overpaint artifacts.
- Current behavior:
  - circles are drawn with track color
  - overlapped parts are masked by higher viaduct area geometry
  - heavily overlapped circles are suppressed for visual stability

Primary file:
- `app.js`

### 2) Track Mode Multi-Level Display and Available Mode Duplication Fixes

- Track edit mode now supports ghost rendering of other levels without dark overdraw.
- Resolved duplication where gradient segments were rendered multiple times in available mode.
- Available mode now uses strict single-level assignment per segment for level-pass rendering.

Primary file:
- `app.js`

### 3) Selection Regression Fixes

- Fixed regressions where track selection looked broken due to overlay condition coupling.
- Restored selection highlight behavior in select mode.
- Changed selected track highlight from dashed centerline to solid frame outline around the segment.
- Adjusted picking fallback so select mode can still pick non-active-level tracks with lower priority.

Primary file:
- `app.js`

### 4) Viaduct Area Erase Compatibility for Legacy Saves

Issue addressed:
- In viaduct area erase mode, newly created areas were erasable, but legacy areas (especially older auto/track-related formats) were not.

Fixes:
- Expanded load-time compatibility for viaduct area formats:
  - `points` as `{x,y}` and `[x,y]`
  - `polygon`
  - level aliases (`level`, `lv`, `trackLevel`)
- Extended erase behavior for non-rect polygons from strict containment-only to overlap-aware removal.

Primary file:
- `app.js`

### 5) Line Color Persistence Compatibility

- Added schema compatibility for both naming conventions:
  - `trackColor`
  - `lineColor`
- Save now writes `lineColor` alongside `trackColor` for compatibility.
- Load now accepts `settings.lineColor` fallback.
- Track normalization also accepts segment color fallbacks (`seg.color`, `seg.trackColor`, `seg.lineColor`).

Primary file:
- `app.js`

### 6) New Line Color Preset System (Named Presets)

Implemented:
- Save current line color with user-defined name.
- Persistent preset storage in localStorage.
- Preset list display and click-to-apply.
- Rename and delete operations for selected preset.
- Confirmation dialogs for:
  - rename commit
  - delete commit

Primary files:
- `app.js`
- `index.html`
- `styles.css`

### 7) Preset UI Refactor to Detached Floating Window

User feedback requested a less space-consuming UX.

Implemented:
- Toolbar section now has:
  - color input
  - `Color` button
- Preset UI moved to a detached floating window.
- Floating window includes:
  - action row (`Save`, `Rename`, `Delete`)
  - on-demand rename/save editor input
  - preset list
- Window can be toggled by `Color` button.
- `Esc` closes the floating window first when open.
- Window supports drag-and-drop movement via handle and non-interactive panel area drag.
- Added robust `[hidden]` CSS handling to prevent state/UI mismatch.

Primary files:
- `app.js`
- `index.html`
- `styles.css`

- `Esc` priority:

  2. otherwise follow existing mode behavior
