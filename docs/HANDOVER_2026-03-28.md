# Handover Memo (2026-03-28)

This memo summarizes the GPU migration phase completed today and the current expected behavior for rendering.

## Completed Today

### 1) GPU-First Rendering Direction Confirmed

- Removed temporary runtime GPU toggle UX and moved to default GPU-first rendering flow.
- Kept CPU fallback paths where appropriate for safety, but main heavy rendering is now routed through GPU.

Primary files:
- app.js
- index.html

### 2) Core GPU Pipeline Expanded

- Added and refined shared GPU geometry submission for:
  - thick line segments (triangle quads)
  - filled circles (triangle fan)
  - filled polygons (triangulated)
  - dashed lines (segment decomposition)
- Unified GPU vertex format to carry color/alpha and masking-related metadata.

Primary file:
- app.js

### 3) Viaduct Mask on GPU Implemented (Mask Texture)

- Added GPU-side mask texture build/update from viaduct area polygons each frame.
- Shader now performs per-fragment masking by comparing area level mask data against primitive base level.
- This removed the major CPU fallback requirement for masked connection circle behavior.

Primary file:
- app.js

### 4) Rendering Targets Migrated to GPU

The following major drawing targets are now GPU-backed:

- Track body
- Connection/intersection circles
- Elevated side rails
- Dead-end markers
- Viaduct wall base rendering
- Viaduct area base fill
- Track clearance (dashed)
- Platform body rendering (segment body + enclosed fill + legacy rectangle fill)
- Train body rendering
- Train selected highlight rendering

Primary file:
- app.js

### 5) Grid Optimization Added (Static Texture Caching)

- Grid drawing was optimized via cached tile pattern generation.
- Pan now mainly shifts phase; expensive dot generation is reused unless zoom-step key changes.
- This is not full GPU geometry rendering, but significantly reduces repeated CPU work.

Primary file:
- app.js
